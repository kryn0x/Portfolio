# Nidhi Kumari — Data Analyst & Visual Data Designer Portfolio

A professional, fully responsive portfolio website showcasing expertise in **Data Analysis × Visual Design × AI Workflows**. Built with pure HTML, CSS, and JavaScript (no frameworks).

**Live Demo:** [View on GitHub Pages](https://kryn0x.github.io/) *(update after deployment)*

---

## About

This portfolio positions Nidhi as a strong candidate for roles at the intersection of **Design, Data, and AI** — specifically targeting positions like **Infographics & Visual Data Design Intern (AI Workflows)**.

---

## Features

### Core Sections
- **Hero**: Updated title "Data Analyst & Visual Data Designer" with AI-focused tagline
- **Skills**: Expanded to include Visual & Design skills (Data Visualization, Visual Hierarchy, Infographic Design, Information Structuring)
- **Infographics & Visual Work**: Showcase section for visual storytelling projects
- **Projects**: Enhanced with "Key Insight" highlights for each project
- **AI & Data Workflow Understanding**: Demonstrates knowledge of data annotation, validation, and AI pipelines
- **What I Bring**: Highlights unique strengths in data-to-visual storytelling
- **Experience**: Clickable internship card with offer letter + certificate modal
- **Education**: Clickable CGPA card linking to report card PDF

### Technical Features
- Dark theme with deep purple-to-blue gradient
- Glassmorphism cards with smooth hover effects
- Typing animation cycling through: Data Analyst, Visual Data Designer, AI Enthusiast, Infographics Designer
- Scroll-reveal animations on all sections
- Animated skill bars with live percentage counters (13 skills total)
- Clickable project cards linking to GitHub repos
- Fully responsive — mobile, tablet, desktop
- Sticky navbar with active section highlighting

## Local Setup

Just open `index.html` in any modern browser — no build step or server required.

---

## Deploy to GitHub Pages

1. Push this entire project to a GitHub repository
2. Go to **Settings → Pages**
3. Under **Source**, select branch: `main`, folder: `/ (root)`
4. Click **Save**
5. Your site will be live at `https://<your-username>.github.io/<repo-name>/`

> All file paths in this project are relative, so everything works both locally and after deployment.
